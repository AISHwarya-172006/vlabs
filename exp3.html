Introduction

In this experiment, we will explore how to design and interface an LCD (Liquid Crystal Display) module with a microcontroller. The LCD will be used to display textual information, which is crucial for providing visual output in embedded systems. By utilizing the GPIO pins of a microcontroller, we can control the LCD and display messages, numbers, or characters on it.

QBJECTIVES

  • Interface LCD Module: Connect the LCD's data and control pins to the microcontroller.

  • Configure GPIO Pins: Set up microcontroller pins to control the LCD.

  • Program Communication Protocol: Use the appropriate commands (e.g., 4-bit or 8-bit mode) to send data to the LCD.

  • Display Data: Write a program to display text or numbers on the LCD.

  • Test and Verify: Ensure that the LCD correctly displays the programmed content.

Theory

An LCD (Liquid Crystal Display) is a flat-panel display that utilizes liquid crystals to display information. It is commonly used in embedded systems to show data such as numbers, characters, or messages. LCD modules come in various sizes and resolutions, but most microcontroller-based systems use the standard 16x2 LCD, which can display two rows of 16 characters.

The interfacing of an LCD with a microcontroller requires proper connection and communication protocols. The communication between the microcontroller and the LCD can be done in either 4-bit or 8-bit mode. The control signals used for interfacing typically include RS (Register Select), RW (Read/Write), and E (Enable), while the data lines carry the actual information to be displayed.

Key Concepts:

  • LCD Pin Configuration:

    The standard 16x2 LCD module has 16 pins. The key pins include:

    • RS (Register Select): Determines whether the operation is for sending data or command.
    • RW (Read/Write): Controls whether the microcontroller is writing data or reading from the LCD.
    • E (Enable): Used to latch the data into the LCD.
    • Data Pins (D0-D7): Used to send data to the LCD in 4-bit or 8-bit mode.
  • Communication Modes:

    The LCD can operate in two modes:

    • 8-bit Mode: All 8 data bits are sent to the LCD in one operation.
    • 4-bit Mode: Only 4 data bits are sent at a time, making it more efficient in terms of pin usage.
  • Initialization:

    The LCD needs to be initialized with specific commands to configure it into the desired mode. This includes setting the data mode (4-bit or 8-bit) and clearing the display.

  • Commands and Functions:

    The LCD operates by sending specific commands that control its behavior, such as moving the cursor, clearing the screen, or changing the display mode. Some common commands include:

    • 0x01: Clear display.
    • 0x80: Set cursor to the beginning of the first line.
    • 0xC0: Set cursor to the beginning of the second line.
    • 0x0C: Display ON, Cursor OFF.

Simulation

Algorithm Steps:

  1. Connect the LCD's data pins (D0-D7) and control pins (RS, RW, EN) to the GPIO pins of the LPC2148.
  2. Configure the GPIO pins as output in the program for proper communication with the LCD.
  3. Initialize the LCD in 8-bit or 4-bit mode and send initialization commands for proper display setup.
  4. Write a program to send data (characters or strings) to the LCD for display.
  5. Use functions to send commands for clearing the display and moving the cursor as required.
  6. Load the program onto the LPC2148 and verify that the LCD displays the intended text or messages.

Source Code

        #include <lpc214x.h>
        #include <stdint.h>    // Pin definitions for LCD
        #define RS (1 << 0)      // P0.0 for Register Select
        #define RW (1 << 1)      // P0.1 for Read/Write
        #define E  (1 << 2)      // P0.2 for Enable
        #define DATA_PINS_MASK (0xF << 4)   // P0.4 to P0.7 for Data
        
        class LCD {
        public:
            LCD() {
                IODIR0 |= (RS | RW | E | DATA_PINS_MASK);  // Set control and data pins as output
                IOCLR0 = (RS | RW | E | DATA_PINS_MASK);   // Clear all pins
                init();
            }    // Initialize the LCD
            
            void init() {
                delay(20);        // Wait for power stabilization
                command(0x02);    // Initialize in 4-bit mode
                command(0x28);    // 2 lines, 5x7 matrix
                command(0x0C);    // Display ON, Cursor OFF
                command(0x06);    // Auto-increment cursor
                command(0x01);    // Clear display
            }
        
            void command(uint8_t cmd) {
                send(cmd, false);
            }    // Send data to the LCD
        
            void write(uint8_t data) {
                send(data, true);
            }    // Display a string on the LCD
        
            void display(const char *str) {
                while (*str) {
                    write(*str++);
                }
            }
        
        private:
            void send(uint8_t value, bool isData) {        // Set RS (data/command mode)
                if (isData) {
                    IOSET0 = RS;
                } else {
                    IOCLR0 = RS;
                }
                IOCLR0 = RW;      // Set RW to Write
                writeNibble(value >> 4);      // Send higher nibble
                writeNibble(value);       // Send lower nibble
            }
        
            void writeNibble(uint8_t nibble) {
                IOCLR0 = DATA_PINS_MASK;  // Clear data pins
                IOSET0 = (nibble & 0xF) << 4;  // Set nibble on data pins
                pulseEnable();  // Send enable pulse
            }
        
            void pulseEnable() {
                IOSET0 = E;
                delay(1);
                IOCLR0 = E;
                delay(1);
            }
        
            void delay(unsigned int ms) {
                for (volatile unsigned int i = 0; i < ms * 3000; i++);
            }
        };
        
        int main() {
            LCD lcd;      // Create an LCD object
            lcd.display("Welcome to Embedded C++");      // Display a message
            while (1) {
                // Keep running
            }
            return 0;
        }
